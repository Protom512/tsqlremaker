# Implementation Tasks: SAP ASE T-SQL Lexer

## Task Format

- `[ ]` 未完了
- `[x]` 完了
- `(P)` 並列実行可能

---

## Phase 1: 基盤インフラストラクチャ

### 1. トークン型システムの実装

- [ ] 1.1 トークン種別の定義 (P)
  - SAP ASE T-SQL のすべてのトークン種別を列挙型として定義する
  - DML キーワード（SELECT, INSERT, UPDATE, DELETE, MERGE）を含める
  - DDL キーワード（CREATE, ALTER, DROP, TRUNCATE）を含める
  - 制御フローキーワード（IF, ELSE, BEGIN, END, WHILE, RETURN, BREAK, CONTINUE）を含める
  - データ型キーワード（INT, VARCHAR, DATETIME, DECIMAL 等）を含める
  - トランザクションキーワード（BEGIN TRAN, COMMIT, ROLLBACK）を含める
  - 演算子トークン（算術、比較、論理、ビット）を含める
  - リテラルトークン（識別子、文字列、数値）を含める
  - 変数と一時テーブルトークン（LocalVar, GlobalVar, TempTable, GlobalTempTable）を含める
  - 型安全とパフォーマンスのために Copy トレイトを実装する
  - _Requirements: 1.1, 1.2, 1.3, 1.4, 6.1, 6.2, 6.3, 6.4, 6.5, 9.1, 9.2, 9.3, 9.4, 9.5, 14.1, 14.2, 14.3, 14.4, 14.5, 15.1, 15.2, 15.3, 15.4, 15.5_

- [ ] 1.2 位置情報構造体の定義 (P)
  - ソースコード上の位置情報（行、列、バイトオフセット）を表す構造体を定義する
  - 1行目、1列目から開始される位置情報を提供する
  - タブ幅を8スペースとして列位置を計算する
  - トークンの開始位置と終了位置を表す範囲（Span）を定義する
  - _Requirements: 7.1, 7.3, 7.4, 13.5_

- [ ] 1.3 静的キーワードマップの構築
  - キーワードの解決に使用する静的な HashMap を定義する
  - プログラム起動時に1回のみ初期化される Lazy 静的初期化を使用する
  - 大文字小文字を区別せずにキーワードを解決する機能を実装する
  - すべての予約語をマップに登録する
  - キーワード解決のパフォーマンス最適化を図る
  - _Requirements: 1.5, 9.5, 11.1_

### 2. カーソルと文字レベルの操作

- [ ] 2.1 文字カーソルの実装
  - 入力文字列を文字単位で走査するカーソルを構築する
  - 現在の文字を取得する機能を提供する
  - 次の文字を先読み（peek）する機能を提供する
  - 2文字先を先読みする機能を提供する
  - 次の文字に進む（bump）機能を提供する
  - CRLF と LF の両方の改行形式を正しく処理する
  - 改行時に行番号と列番号を正しく更新する
  - タブ文字を8スペースとして列位置を計算する
  - 現在の位置情報を取得する機能を提供する
  - EOF 状態を判定する機能を提供する
  - _Requirements: 7.1, 7.2, 7.3, 7.4, 13.2, 13.4, 13.5_

## Phase 2: エラーハンドリング

### 3. エラー型システム

- [ ] 3.1 字句解析エラー型の定義
  - 終了していない文字列リテラルを表すエラー型を定義する
  - 終了していないブロックコメントを表すエラー型を定義する
  - 終了していない引用符付き識別子を表すエラー型を定義する
  - 不正な文字を表すエラー型を定義する
  - 予期しない EOF を表すエラー型を定義する
  - エラーの発生位置を行番号と列番号で示す機能を実装する
  - エラーメッセージを生成する機能を実装する
  - エラー位置を取得する機能を実装する
  - _Requirements: 2.4, 4.5, 8.1, 8.2, 8.4, 10.4_

## Phase 3: コメント処理

### 4. コメントの字句解析

- [ ] 4.1 ブロックコメントの読み取り
  - ブロックコメント `/* */` を認識してトークンを生成する機能を実装する
  - ブロックコメント内に別のブロックコメントがネストされている場合の深さを追跡する
  - ネストの深さを正しく管理してコメントの終了を判定する
  - コメントが終了していない状態で EOF に到達した場合のエラー検出を実装する
  - コメントを保持するオプションに対応する
  - _Requirements: 2.1, 2.3, 2.4, 2.5_

- [ ] 4.2 ラインコメントの読み取り (P)
  - ラインコメント `--` を認識してトークンを生成する機能を実装する
  - 改行までをコメントとして読み取る機能を実装する
  - コメントを保持するオプションに対応する
  - _Requirements: 2.2, 2.5_

## Phase 4: 変数と一時テーブル

### 5. 変数トークンの処理

- [ ] 5.1 @ 変数プレフィックスの読み取り
  - `@` プレフィックスを持つ識別子を認識して LocalVar トークンを生成する
  - `@@` プレフィックスを持つ識別子を認識して GlobalVar トークンを生成する
  - 変数名の部分を値として保持する機能を実装する
  - グローバル変数名を大文字小文字を区別せずに認識する
  - _Requirements: 3.1, 3.2, 3.5, 14.5_

- [ ] 5.2 # 一時テーブルプレフィックスの読み取り (P)
  - `#` プレフィックスを持つ識別子を認識して TempTable トークンを生成する
  - `##` プレフィックスを持つ識別子を認識して GlobalTempTable トークンを生成する
  - 識別子名の部分を値として保持する機能を実装する
  - _Requirements: 3.3, 3.4, 3.5_

## Phase 5: 文字列リテラル

### 6. 文字列の字句解析

- [ ] 6.1 通常文字列リテラルの読み取り
  - シングルクォートで囲まれた文字列を認識して String トークンを生成する
  - 文字列内にエスケープされたクォート（`''`）を文字列の一部として処理する
  - 文字列リテラルが終了していない状態で EOF に到達した場合のエラー検出を実装する
  - _Requirements: 4.1, 4.4, 4.5_

- [ ] 6.2 Unicode 文字列の読み取り (P)
  - `N'...'` 形式の Unicode 文字列を認識して NString トークンを生成する
  - `U&'...'` 形式の Unicode エスケープ文字列を認識して UnicodeString トークンを生成する
  - Unicode エスケープシーケンスを正しく処理する
  - エスケープされた引用符を処理する
  - _Requirements: 4.2, 4.3, 4.4, 4.5_

## Phase 6: 数値リテラル

### 7. 数値の字句解析

- [ ] 7.1 数値リテラルの読み取り
  - 整数リテラルを認識して Number トークンを生成する
  - 小数点を含む数値を認識して Float トークンを生成する
  - 科学表記法（`1.5e10` など）の数値を認識して Float トークンを生成する
  - `0x` プレフィックスを持つ16進数を認識して HexString トークンを生成する
  - 数値リテラルの元の文字列表現を値として保持する
  - _Requirements: 5.1, 5.2, 5.3, 5.4, 5.5_

## Phase 7: 演算子と区切り文字

### 8. 演算子の字句解析

- [ ] 8.1 算術演算子の読み取り (P)
  - 算術演算子（`+`, `-`, `*`, `/`, `%`）を認識して対応するトークンを生成する
  - 複合代入演算子（`+=`, `-=`, `*=`, `/=`）を認識して対応するトークンを生成する
  - _Requirements: 6.1, 15.2_

- [ ] 8.2 比較演算子の読み取り (P)
  - 比較演算子（`=`, `<>`, `!=`, `<`, `>`, `<=`, `>=`, `!<`, `!>`）を認識して対応するトークンを生成する
  - ASE 固有の比較演算子（`!<`, `!>`）を含める
  - _Requirements: 6.2, 15.5_

- [ ] 8.3 その他の演算子の読み取り (P)
  - ビット演算子（`&`, `|`, `^`, `~`）を認識して対応するトークンを生成する
  - 論理演算子（`AND`, `OR`, `NOT`）をキーワードとして認識する
  - 文字列連結演算子（`||`）を認識して Concat トークンを生成する
  - 範囲演算子 `..` を認識して DotDot トークンを生成する
  - ドット `.` を認識して Dot トークンを生成する
  - _Requirements: 6.3, 6.4, 6.5, 15.3, 15.4_

- [ ] 8.4 区切り文字の読み取り (P)
  - 括弧（`(`, `)`, `{`, `}`）を認識して対応するトークンを生成する
  - 角括弧（`[`, `]`）を認識して対応するトークンを生成する
  - カンマ、セミコロン、コロンを認識して対応するトークンを生成する
  - _Requirements: 1.4_

## Phase 8: 引用符付き識別子

### 9. 引用符付き識別子の字句解析

- [ ] 9.1 角括弧付き識別子の読み取り (P)
  - 角括弧 `[...]` で囲まれた識別子を認識して QuotedIdent トークンを生成する
  - 引用符付き識別子内にエスケープされた閉じ括弧（`]`）を識別子の一部として処理する
  - 引用符付き識別子が終了していない場合のエラー検出を実装する
  - 引用符を除いた識別子名を値として保持する
  - _Requirements: 10.1, 10.3, 10.4, 10.5_

- [ ] 9.2 二重引用符付き識別子の読み取り (P)
  - 二重引用符 `"..."` で囲まれた識別子を認識して QuotedIdent トークンを生成する
  - 二重引用符内のエスケープ処理を実装する
  - 引用符を除いた識別子名を値として保持する
  - _Requirements: 10.2, 10.3, 10.4, 10.5_

## Phase 9: 識別子とキーワード

### 10. 識別子とキーワードの字句解析

- [ ] 10.1 識別子とキーワードの読み取り
  - アルファベットまたはアンダースコアで始まる識別子を認識する
  - 識別子の連続する文字を読み取る機能を実装する
  - 静的キーワードマップを使用して識別子をキーワードに解決する
  - 大文字小文字を区別せずにキーワードを認識する
  - 予約語でない識別子の場合は Ident トークンを生成する
  - _Requirements: 1.1, 1.2, 1.5_

## Phase 10: 空白と改行の処理

### 11. 空白の字句解析

- [ ] 11.1 空白と改行のスキップ
  - 空白文字をスキップして次のトークンを生成する機能を実装する
  - 改行文字が入力された場合に行番号と列番号を正しく更新する
  - コメント保持オプションが有効でない場合にコメントをスキップする
  - CRLF と LF の両方の改行形式を正しく処理する
  - _Requirements: 7.3, 13.1, 13.2, 13.3, 13.4_

## Phase 11: Lexer 本体の実装

### 12. Lexer メイン構造体

- [ ] 12.1 Lexer 構造体の実装
  - 入力文字列を受け取って新しい Lexer を作成する機能を実装する
  - コメントを保持するか設定する機能を実装する
  - 次のトークンを取得するメインループを実装する
  - EOF に到達した場合に EOF トークンを返す機能を実装する
  - すべてのトークン種別のディスパッチロジックを実装する
  - _Requirements: 1.1, 1.2, 1.3, 1.4, 1.5, 12.1, 12.2_

- [ ] 12.2 エラーリカバリの実装
  - トークンとして認識できない不正な文字が入力された場合に Unknown トークンを生成してエラーを報告する
  - エラーが発生した場合に次の同期ポイント（セミコロンまたはキーワード）まで読み取りを続けて処理を継続する
  - エラーを収集して後でまとめて報告する機能を実装する
  - エラーがあるかどうかを判定する機能を実装する
  - _Requirements: 8.1, 8.3, 8.5, 12.5_

### 13. Zero-copy トークン構造体

- [ ] 13.1 Token 構造体の実装
  - ソースコードへの参照（ゼロコピー）を使用してトークンを表現する構造体を定義する
  - トークン種別、テキスト、スパンを含むトークンを構築する
  - 不要な文字列割り当てを回避する
  - _Requirements: 7.1, 11.3, 11.4_

## Phase 12: イテレータと先読み機能

### 14. パーサー統合機能

- [ ] 14.1 イテレータの実装
  - イテレータ可能なトークンストリームを提供するために Iterator トレイトを実装する
  - すべてのトークンが消費された場合に None を返すようにする
  - _Requirements: 12.1, 12.3_

- [ ] 14.2 先読み（peek）機能の実装 (P)
  - 次のトークンを消費せずに取得する peek 機能を実装する
  - 複数のトークンを先読みできるようにする
  - Parser がエラーを通知した場合に次の同期ポイントまでスキップするための情報を提供する
  - _Requirements: 12.2, 12.4, 12.5_

### 15. パフォーマンス機能の実装

- [ ] 15.1 パフォーマンス最適化の実装
  - 静的 HashMap を使用してキーワード解決の再作成オーバーヘッドを回避する

  - ブロックコメント（単純）のテストを実装する
  - ネストされたブロックコメントのテストを実装する
  - ラインコメントのテストを実装する
  - 終了していないコメントのエラー検出のテストを実装する
  - ローカル変数 `@var` のテストを実装する
  - グローバル変数 `@@var` のテストを実装する
  - 一時テーブル `#table` のテストを実装する
  - グローバル一時テーブル `##table` のテストを実装する
  - 通常文字列リテラルのテストを実装する

  - Unicode 文字列 `N'...'` のテストを実装する

  - 終了していない文字列のエラー検出のテストを実装する
  - 算術演算子のテストを実装する
  - 比較演算子のテストを実装する
  - ビット演算子のテストを実装する
  - 代入演算子と複合代入演算子のテストを実装する
  - 文字列連結演算子 `||` のテストを実装する
  - 行番号の正確な追跡のテストを実装する
  - 列番号の正確な追跡（タブを考慮）のテストを実装する
  - 改行形式（CRLF/LF）の両方に対応するテストを実装する
  - _Requirements: 1.1, 1.2, 1.3, 1.4, 1.5, 2.1, 2.2, 2.3, 2.4, 3.1, 3.2, 3.3, 3.4, 4.1, 4.2, 4.3, 4.4, 5.1, 5.2, 5.3, 5.4, 6.1, 6.2, 6.3, 6.4, 6.5, 7.3, 7.4, 13.2, 13.4, 13.5_

  - CREATE PROCEDURE 文のトークン化テストを実装する
  - 複雑な JOIN を含むクエリのトークン化テストを実装する
  - エラーを含む SQL のリカバリテストを実装する
  - _Requirements: 1.1, 8.1, 8.5_

- [ ] 16.3 パフォーマンステストの実装 (P)
  - 1MB の SQL ファイルを 100ms 以下で処理するベンチマークを実装する
  - キーワード解決の HashMap ヒット率を検証する
  - Zero-copy のメモリ効率を検証する
  - _Requirements: 11.1, 11.2, 11.3_
