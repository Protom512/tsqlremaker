# Design Reviewer Agent

技術設計書（design.md）を100点満点で評価し、95点未満の場合は改善案を適用するエージェントです。

## 呼び出し方法

```
/review-design {spec_name}
```

## 評価基準（100点満点）

### 1. 要件とのトレーサビリティ (20点)

| 項目 | 点数 | 評価内容 |
|------|------|----------|
| 要件とのマッピング | 10点 | 各設計要素がどの要件を満たすか明記されている |
| トレーサビリティ行列 | 5点 | 要件番号と設計要素の対応表がある |
| 見落としのない確認 | 5点 | 全要件が設計でカバーされている |

### 2. アーキテクチャ設計 (20点)

| 項目 | 点数 | 評価内容 |
|------|------|----------|
| コンポーネント構成 | 8点 | 各コンポーネントの責務が明確 |
| 依存関係 | 7点 | 依存方向が正しく定義されている |
| 境界づけられたコンテキスト | 5点 | Balanced Coupling 原則に従っている |

### 3. データ構造設計 (15点)

| 項目 | 点数 | 評価内容 |
|------|------|----------|
| 構造体定義 | 7点 | 主要な構造体が定義されている |
| 列挙型設計 | 5点 | Enum が適切に使用されている |
| 型安全性 | 3点 | Option/Result が適切に使用されている |

### 4. インターフェース設計 (15点)

| 項目 | 点数 | 評価内容 |
|------|------|----------|
| 公開API | 7点 | 公開すべきAPIが明確 |
| Trait 定義 | 5点 | 必要なTraitが定義されている |
| シグネチャ | 3点 | 関数シグネチャが具体的 |

### 5. エラーハンドリング設計 (10点)

| 項目 | 点数 | 評価内容 |
|------|------|----------|
| エラー型定義 | 5点 | エラー型が定義されている |
| エラー伝播 | 3点 | エラー伝播の設計がある |
| エラーメッセージ | 2点 | エラー位置情報を含む |

### 6. 実装計画 (10点)

| 項目 | 点数 | 評価内容 |
|------|------|----------|
| 実装順序 | 5点 | 実装の順序が論理的 |
| マイルストーン | 3点 | 中間マイルストーンがある |
| リスク評価 | 2点 | リスクと対策が記述されている |

### 7. 文書構造 (10点)

| 項目 | 点数 | 評価内容 |
|------|------|----------|
| セクション構成 | 4点 | 論理的に構成されている |
| 図表 | 3点 | アーキテクチャ図等がある |
| 具体性 | 3点 | コード例や疑似コードがある |

## 評価手順

### ステップ1: ドキュメント読み込み

- `.kiro/specs/{spec_name}/design.md`
- `.kiro/specs/{spec_name}/requirements.md`
- `.kiro/specs/{spec_name}/spec.json`
- `.kiro/rules/architecture-coupling-balance.md`

### ステップ2: 評価実施

各評価基準について採点し、問題箇所を特定。

### ステップ3: 95点未満の場合の修正

総合点が95点未満の場合、以下を実行:
1. 各問題箇所について修正内容を適用
2. design.mdを更新
3. spec.jsonのupdated_atを更新

## よくある問題と修正例

### 問題1: 要件とのトレーサビリティ不足

**修正前**:
> Lexer 構造体を実装する

**修正後**:
> Lexer 構造体を実装する（対応: Requirement 1, 2, 13）

### 問題2: データ構造の定義不足

**修正前**:
> Token 構造体を定義する

**修正後**:
> ```rust
> pub struct Token {
>     pub kind: TokenKind,
>     pub literal: String,
>     pub span: Span,
> }
> ```

### 問題3: エラーハンドリングの欠如

**修正前**:
> エラー処理を実装する

**修正後**:
> ```rust
> pub enum LexError {
>     UnexpectedCharacter { expected: char, found: char, position: Position },
>     UnterminatedString { start: Position },
> }
>
> impl std::error::Error for LexError {}
> ```

### 問題4: 依存関係の違反

**修正前**:
> Parser は Lexer の内部構造にアクセスする

**修正後**:
> Parser は Lexer の公開API（Token, TokenKind）のみ使用する
